# Contributing to Restaurant SAAS Backend

Thank you for your interest in contributing! This document outlines the process for contributing to the project.

## How to Contribute

1. **Fork the repository** and clone your fork locally.
2. **Create a new branch** for your feature or bugfix:
   ```sh
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes** following the project structure and coding standards.
4. **Write tests** for your changes (unit, integration, or e2e as appropriate).
5. **Run tests locally** to ensure nothing is broken:
   ```sh
   npm test
   ```
6. **Commit your changes** with clear messages:
   ```sh
   git commit -m "Add feature: your-feature-name"
   ```
7. **Push your branch** to your fork and open a Pull Request (PR) against the `qa` branch.
8. **Describe your changes** clearly in the PR description.

## Code Style

- Use TypeScript and follow the existing code conventions.
- Organize code by modules and core utilities.
- Write clear, maintainable, and well-documented code.

## Testing

- Add or update tests in the `tests/` directory.
- Ensure all tests pass before submitting a PR.

## Reporting Issues

- Use GitHub Issues to report bugs or request features.
- Provide as much detail as possible, including steps to reproduce and expected behavior.

## Contact

For questions, reach out to the repository owner or open an issue.
